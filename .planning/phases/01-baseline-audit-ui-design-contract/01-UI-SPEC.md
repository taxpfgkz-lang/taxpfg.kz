# Phase 1 — UI Design Contract

> Visual and interaction contract for the taxpfg.kz Production UI Polish milestone.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** taxpfg.kz is a no-build static marketing site (11 HTML pages) on the
> GudFin/PBMIT theme + Bootstrap 5.2, with a thin `pfg-*` override layer. The stack is
> frozen and vendor-read-only — there is NO component library or styling stack to choose.
> This document is the **formalized design contract (deliverable AUD-04)**: it fixes the
> existing tokens, spacing scale, type scale, color+contrast floor, component states, and
> hard constraints that Phases 2–5 implement against. It does not introduce new tooling.

| Property | Value |

## Spacing Scale

Token-based 4px scale, declared as `--pfg-space-*` (Phase 2 / TOK-02). All values are multiples of 4. Existing theme section paddings are **snapped to the nearest token** — not fully recalculated — to preserve the visual rhythm.

| Token | Value | Usage |
|-------|-------|-------|
| `--pfg-space-1` | 4px | Icon gaps, inline hairline padding |
| `--pfg-space-2` | 8px | Compact element spacing, tight stacks |
| `--pfg-space-3` | 12px | Eyebrow gap (existing `gap:12px`), small control padding |
| `--pfg-space-4` | 16px | Default element spacing, form-field gaps |
| `--pfg-space-6` | 24px | Card padding, section inner padding |
| `--pfg-space-8` | 32px | Layout gaps, block separation |
| `--pfg-space-12` | 48px | Major section breaks |
| `--pfg-space-16` | 64px | Page-level section spacing |
| `--pfg-space-24` | 96px | Hero / large section vertical rhythm |

**Exceptions:**
- Interactive touch targets keep a **44px minimum hit zone** (WCAG 2.5.5) even when the visual glyph is smaller — see `css/custom.css:164-170`, footer menu `:565-571`. This is a hit-area floor, not a spacing-scale value.
- Vendor theme paddings that already read correctly are snapped to the nearest token; do not break the existing rhythm to force an exact scale value.

## Typography

Fluid `clamp()` type scale, ratio ≈ 1.25 (major third), declared in Phase 2 (TOK-03). Body base 16–18px (current theme body = 17px, `base.css:37`). Heading family Plus Jakarta Sans (weight 600); body family Be Vietnam Pro (weight 400/regular). **Exactly two weights**: regular (400) + semibold (600).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | clamp(16px → 18px) | 400 | 1.6 (existing `--pbmit-body-typography-line-height`) |
| Label / UI | 15px (existing `--pbmit-btn-typography-font-size`) | 600 | 1.2 |
| Heading (h2–h4) | clamp(20px → ~34px), ratio 1.25 | 600 | 1.2 |
| Display (h1 / hero) | clamp(~34px → ~56px) | 600 | 1.15–1.2 |

**Wrapping & Russian-language rules:**
- `text-wrap: balance` on headings; `text-wrap: pretty` on body — orphan/widow guard.
- Non-breaking handling for short Russian prepositions/conjunctions (`в`, `и`, `с`, `к`, `на`, `по`, `от`) so they never end a line.
- `text-transform: none` wherever the theme imposes `capitalize`/`uppercase` on Russian text — rely on correct HTML casing (existing pattern: `custom.css:215-218`, `:243-258`, `:865-873`). The theme's transforms mangle Russian (capitalize prepositions).
- Exactly one `<h1>` per page; index uses visually-hidden `<h1 class="pfg-sr-only">`. Markup is read-only this milestone — do not change heading levels.

## Color

Ink + gold palette. **Palette is frozen — not expanded.** Brand hex values are reused from `--pbmit-*`; only *text* color or *text-on-gold* is darkened for contrast, never the brand fill.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f6f4ef` warm paper (`--pfg-paper-warm`) + `#ffffff` | Page background, surfaces (replaces theme's cold `#ecf0f4`) |
| Secondary (30%) | `#16222d` ink (`--pfg-ink`), `#0f1820` ink-deep (`--pfg-ink-deep`) | Footer, dark sections, headings, ink-on-light text |
| Accent (10%) | `#ecab23` gold (`--pfg-gold`) | See reserved-for list below |
| Hover accent | `#d6960f` gold-deep (`--pfg-gold-deep`) | Gold hover state (text `#1b1b1b` holds AA on it) |
| Gold-as-text | `#7a560a` gold-ink (`--pfg-gold-ink`) | Gold-colored *text* on light surfaces only (AA ≥ 5.4:1) |
| Hairlines | `rgba(22,34,45,.10)` / `.16` | Borders, dividers (`--pfg-hairline`, `--pfg-hairline-2`) |
| Body text | `#7c898d` → darkened for AA (`custom.css:122`) | Default body copy |

**Accent reserved for** (gold fill `#ecab23` — explicit list, never "all interactive elements"):
- Primary CTA button fill
- Eyebrow "ledger stroke" mark before subtitles (`::before`, `custom.css:347+`)
- Footer 3px gold top border (`custom.css:512`)
- Focus-visible ring (`outline: 2px solid var(--pfg-gold)`, `custom.css:307-314`)
- Active-link / hover accent on nav (theme `--pbmit-link-color-hover`)
- The single "популярный" pricing-tier highlight (Phase 4, CNV-02)

**Destructive:** No destructive/delete actions exist on this site (lead-gen marketing site, no account/data mutations). Form-error state uses a non-gold warning treatment via `.pfg-form-status.is-error` (text + message, not a color-only signal).

**Contrast floor (WCAG AA):**
- Body / normal text ≥ 4.5:1
- Large text and UI components / graphics ≥ 3:1
- Gold (`#ecab23`) is **forbidden as body text** on light backgrounds — use `--pfg-gold-ink` (`#7a560a`) instead. Gold fill stays `#ecab23`; only text color changes.

## Component State Contract

All interactive elements MUST declare **hover / focus-visible / active / disabled**. Focus is **always visible** — no `outline:none` without a replacement (A11Y-01). Reference focus ring: `outline: 2px solid var(--pfg-gold); outline-offset: 3px` (`custom.css:307-314`).

| Component | States required | Notes |
|-----------|-----------------|-------|
| Buttons | primary / secondary / ghost hierarchy × hover/focus-visible/active/disabled | `.pbmit-btn` is the theme button; style via override, never rename. Hover uses `--pfg-gold-deep`; shadows `--pfg-shadow-btn` / `-btn-h` |
| Forms / inputs | rest / focus / error / success | Real `<label>`, `autocomplete`, `required`, `novalidate` (JS owns validation). Soft gold focus ring (`custom.css` 9.9). Form→WhatsApp behavior unchanged (CMP-02) |
| Service cards (`.pfg-card`) | rest / hover (lift) | Hover lift is decorative → must be in scoped reduced-motion block |
| Nav (sticky header + off-canvas) | rest / hover (underline grow) / active / focus-visible | Glass header gated `min-width:1201px`. Underline grow is decorative motion |
| FAQ accordion | collapsed / expanded / focus | Correct ARIA + keyboard (CMP-05) |
| Modals (Magnific Popup) | open / focus-trap | **Only if Magnific Popup is actually instantiated** — confirm in Phase 3 before writing rules (CMP-06) |

**Button hierarchy (CMP-01):**
- **Primary** — gold fill `#ecab23`, ink/dark text holding AA, `--pfg-shadow-btn`. The single prominent action per view.
- **Secondary** — outline/ink variant (e.g. `.pbmit-btn white` on dark sections).
- **Ghost** — text-weight action, minimal chrome, gold-ink on hover.

## Hard Constraints (binding on Phases 2–5)

These are contract-level guardrails the planner and executor MUST honor.

| Constraint | Rule | Source |
|------------|------|--------|
| visual-only | No business-logic / form-flow / analytics / routing / JSON-LD behavior changes; presentation only | PROJECT.md, VER-04 |
| vendor read-only | Edit ONLY `css/custom.css`, `css/base.css`, `js/custom.js`. All other `css/*` and `js/*` are theme-owned | CLAUDE.md, `custom.css:1-4` |
| no `@layer` | CSS `@layer` is FORBIDDEN — unlayered vendor always beats layered. Win by source-order + targeted specificity only | REQUIREMENTS Out-of-Scope, research HIGH |
| focus-always | Every interactive element keeps a visible `:focus-visible` indicator; no bare `outline:none` | A11Y-01 |
| scoped-motion | New decorative motion goes inside the scoped `prefers-reduced-motion` block (`custom.css:332-345`); brand motion (Swiper/marquee/GSAP/AOS) floor stays intact — never use a universal `*` killer | A11Y-03, `custom.css:315-331` |
| no-build | No bundler/npm/build step; everything works by opening HTML in a browser | PROJECT.md |
| load-order | `custom.css` stays the LAST stylesheet; `custom.js` the LAST script — or overrides stop winning | `index.html:54`, `:1057` |
| off-canvas-safe | No `filter`/`backdrop-filter`/`transform` on ancestors of the mobile off-canvas menu (collapses `height:100%`). Glass-header effects gated to `min-width:1201px` | `custom.css:382-390` |
| change-all-11 | Shared chrome (header/footer/nav/scripts) is duplicated across 11 HTML files — any shared markup edit applies to all 11 (grep returns 11) | CLAUDE.md |
| `!important` budget | ~59 baseline. Net-new ≈ 0; each new one must cite the vendor rule it beats in a comment | ROADMAP P2 notes |
| comments-russian | New `custom.css`/`custom.js` comments are Russian, dense, justify *why* (vendor behavior, measured contrast, WCAG criterion) | CLAUDE.md |
