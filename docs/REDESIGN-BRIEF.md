# Редизайн taxpfg.kz без изменения текста — конверсионное ТЗ для дизайнера

> **Контекст.** Документ подготовлен по живому сайту (прочитан HTML/CSS всех ключевых
> страниц + отрендерены экраны desktop 1440 / mobile 390 через Playwright), а не по
> скриншотам. Сайт уже несёт премиальный слой «чернила + золото» поверх темы GudFin
> (`css/custom.css`, 1413 строк, Этапы 8–16). Поэтому это **не ре-скин с нуля**, а
> эволюция готовой системы в сторону продаж под рекламный трафик.
>
> **Жёсткие ограничения проекта (не нарушать):**
> - Текст утверждён — не переписывать, не сокращать, не менять офферы/заголовки/кнопки.
> - Вендорные файлы темы read-only. Правки только: `css/custom.css`, `css/base.css`, `js/custom.js`.
> - No-build: всё работает открытием HTML; бандлер вводить нельзя.
> - Логика форм (→ WhatsApp), аналитика, JSON-LD, роутинг — поведение не меняем.
> - 11 HTML-страниц без include: общие правки шапки/футера вносятся во все 11 руками.
> - Бренд-палитра заморожена: ink `#16222d`, gold `#ecab23`. Floor доступности — WCAG AA.
> - HTML-разметку править можно (markup-read-only снят), но осторожно и одинаково на всех 11.

## 2. Главная стратегия редизайна

**Тезис:** «Спокойная дорогая бухгалтерия, которая снимает тревогу за налоги и сроки».
Не «дёшево и быстро», а «надёжно, по закону, без штрафов». Премиум-доверие, не distress-маркетинг.

**Принципы:**
1. **Эволюция, не революция.** Палитра/типографика/сигнатура остаются. Меняем композицию,
   иерархию, плотность доверия и хореографию CTA.
2. **Один экран — одно действие.** На каждом «складе внимания» ровно один заметный
   primary-CTA (золото). Вторичные — ghost/outline. Уже заложено токенами; распространить.
3. **Доверие — визуальный слой, а не абзац.** Реальные факты (с 2013, 1С, по законам РК,
   конфиденциально, договор) вынести в бейджи/строку доверия. Выдуманное (счётчики/отзывы)
   — НЕ добавлять; зарезервировать слоты под данные клиента.
4. **Мобайл-first для трафика.** Телефон + WhatsApp всегда в зоне большого пальца.
5. **Золото — драгоценность, не краска.** Золото только на действиях и одном акценте на
   экран. Всё остальное — ink на бумаге. (Совет Chanel: «снять одно украшение».)

**Визуальное настроение:** строгий fintech-консалтинг; «бухгалтерская книга»: hairline-линии,
табличный ритм, золотой «штрих учёта» как лейтмотив. Уровень премиальности — высокий, но
сдержанный (банк, не лакшери-бутик).

**Цветовая логика (не расширять, закрепить роли):**
- `ink #16222d` / `ink-deep #0f1820` — текст, тёмные секции, marquee-фон.
- `gold #ecab23` — ТОЛЬКО action/один акцент; `gold-deep #d6960f` hover; `gold-ink #7a560a` золото-текст на светлом (AA).
- `paper-warm #f6f4ef` — чередующиеся секции; белый — основной фон.
- hairline `rgba(22,34,45,.10/.16)` — границы карточек, разделители.

**Типографика (уже подключена, использовать осознанно):** Plus Jakarta Sans — заголовки/кнопки
(характер); Be Vietnam Pro — body. Шкала clamp() `--pfg-fs-*` уже есть. Усилить: больший
контраст веса (display 700–800 vs body 400), мера строки 60–72ch (уже ограничена в §10.1).

**Сетка/композиция:** Bootstrap 12-кол, контейнер ≤1200–1320px. Ритм секций 80/56/44 (desktop/
tablet/mobile) — эталон, уже внедрён. Принцип отступов: 4px-сетка `--pfg-space-*`. Z-паттерн на
hero, F-паттерн на контентных.

**Изображения/иконки:** иконки — текущие линейные SVG с золотым акцентом (ок). Фото — заменить
generic-сток на «казахстанский деловой» контекст (рабочее место бухгалтера, документы РК,
Алматы) с единым ink-grade-фильтром (тёплый дуотон ink+gold) для консистентности.

## 4. Первый экран (hero) — главный приёмник трафика

**Текущее состояние:** fade-слайдер из 3 слайдов, на каждом — крупный белый заголовок (2 слова),
вторичная белая кнопка («Наши услуги»/«Подробнее»), подпись-оффер прижата в правый нижний угол
(обрезается на desktop, исчезает на mobile). Золотой CTA — только в шапке. Generic-сток-фото.

**Цель экрана:** за 3 секунды сказать «кто/что/для кого/почему вам» и дать одно сильное действие.

**Композиция (рекомендованная — вариант A, левый-выровненный Z-паттерн):**
```
┌───────────────────────────────────────────────────────────────┐
│  [шапка: лого | тел+WhatsApp | меню | Получить консультацию]   │
├───────────────────────────────────────────────────────────────┤
│  ▏[надзаголовок: Бухгалтерия под ключ / город Алматы]          │
│                                                                │
│  [H1: текущий крупный заголовок слайда]            [фото-      │
│  [подзаголовок-оффер: «Ведём 1С, сдаём отчётность, решаем      │
│   проблемы с госорганами — рутину берём на себя»]   дуотон     │
│                                                     ink+gold]  │
│  [🟡 Получить консультацию ]  [⚪ Наши услуги]                  │
│                                                                │
│  ▏с 2013 · 1С · по законам РК · конфиденциально  (строка       │
│                                              доверия-чипы)     │
└───────────────────────────────────────────────────────────────┘
```

**Решения:**
- **Заголовок + оффер вместе, слева, в читаемой зоне** (чинит D3). Подпись-оффер больше не
  в углу — она подзаголовок под H1.
- **Primary-CTA в hero — золото** «Получить консультацию» (текст уже есть в шапке — переиспользуем
  ту же копию), рядом ghost «Наши услуги». Чинит D2. (CNV-01 в CSS уже гарантирует один primary на слайд.)
- **Строка доверия** из реальных фактов под CTA (чинит D6 без выдумок).
- **Слайдер (D4):** оставить 3 слайда как визуальный ритм, НО зафиксировать единый
  подзаголовок-оффер и единый primary-CTA на всех слайдах (меняется только H1+фон). Либо —
  тест: статичный hero под главный оффер «Бухгалтерия под ключ» (см. варианты ниже).
- **Фото (D8):** заменить сток на казахстанский деловой контекст, наложить ink-дуотон-градиент
  (уже есть `.pbmit-title-bar` паттерн) — текст читается, фото не спорит с типографикой.
- **Усиление кнопки:** золото `#ecab23`, ink-текст, тень `--pfg-shadow-btn`, hover-подъём
  (уже в §9.5). Размер тапа ≥48px, на mobile — ширина по контенту, не во весь экран.

**Mobile-адаптация first screen:**
- Шапка: лого + 📞 + WhatsApp + гамбургер (D1).
- H1 (clamp 34–52px уже есть) → подзаголовок-оффер (вернуть на mobile, сейчас исчезает) →
  золотой CTA во всю шир-минус-поля → строка доверия в 2 строки чипами.
- Высота hero ≤ 80vh, чтобы был «выглядывающий» следующий блок (affordance скролла).

**2–3 варианта для A/B-теста (одинаковый текст, разная подача):**
- **A — «Оффер слева + доверие»** (выше): максимум смысла, классический перформанс-лендинг.
- **B — «Статичный фокус-оффер»:** один экран, один оффер «Бухгалтерия под ключ», крупный
  H1 + подзаголовок + золотой CTA + строка доверия, фон — спокойное фото с сильным дуотоном.
  Гипотеза: меньше расфокуса (чинит D4 радикально) → выше CR с холодного трафика.
- **C — «Сплит с мини-формой»:** слева оффер+доверие, справа — компактная форма заявки
  (имя+телефон+кнопка, копия из contacts). Гипотеза: лид без перехода на /contacts (чинит D5).

## 5. UI-kit и визуальная система (production-ready)

> Источник истины — токены `--pfg-*` в `css/custom.css:300+`. Ниже — что закрепить/дорисовать.

**Цвета (роли, не расширять палитру):**
- Surface: `#ffffff`, alt `#f6f4ef`, ink-секции `#16222d`/`#0f1820`.
- Text: основной `#525d62` (AA), заголовки `#16222d`, золото-текст `#7a560a`.
- Action: gold `#ecab23` (rest) → `#d6960f` (hover), текст на золоте `#1b1b1b` (AA 8.5:1).
- Lines: hairline `rgba(22,34,45,.10)` / `.16`.

**Типографика:**
- Display (Plus Jakarta Sans 700–800): H1 clamp→58, H2→48, H3→40 (`--pfg-fs-*`).
- Body (Be Vietnam Pro 400/600): 16→18px, lh 1.6, мера 60ch.
- Eyebrow: золото-штрих + 700, letter-spacing 1.4px, `text-transform:none` (кириллица!).
- Кнопки/метки: 15px, `text-transform:none` (обязательно — тема ломает русские предлоги).

**Кнопки (3 тира — уже в CSS, закрепить в макете):**
- Primary: золотая заливка, ink-текст, radius 8px, тень, hover-подъём −2px.
- Secondary: `.white`/`.blackish` (тёмные секции).
- Ghost: прозрачный, золото-текст, hairline-рамка, hover — мягкая золотая подложка.
- Состояния: hover/focus-visible(золотое кольцо 2px/offset 3px)/active/disabled(opacity .55).

**Карточки:** белый фон, hairline, radius 14px, тень-sm, hover: подъём −4px + тень-md + золотая
рамка. Внутр. отступ 24px (4px-сетка). Заголовок lh 1.3, `text-wrap:balance`.

**Формы:** поля 16px (анти-iOS-zoom), hairline-рамка, radius 10px, фокус — золотая рамка +
кольцо `rgba(236,171,35,.18)`. Чекбокс 18px золото. Статус: ⚠/✓ глиф + цветная левая рамка
(не только цвет — WCAG 1.4.1). Лейбл — настоящий `<label>`, `novalidate` (JS-валидация).

**Бейджи/чипы:** золотой бейдж «Популярный» (ink-текст); чипы доверия — hairline-контур,
ink-текст, золотая точка-маркер.

**Фоны:** белый / тёплая бумага / ink-градиент (title-bar, hero, process). Дуотон-фильтр
ink+gold для всех контентных фото — единый грейд.

**Тени/радиусы/отступы:** шкалы `--pfg-shadow-sm/md/btn`, `--pfg-radius 14 / sm 10 / кнопки 8`,
`--pfg-space-*` (4px). Easing `--pfg-ease cubic-bezier(.22,.61,.36,1)`.

**Сигнатура (фирменный элемент):** золотой «штрих учёта» (ledger stroke) перед надзаголовками +
3px золотая линия над футером. Это узнаваемый элемент — переиспользовать, не множить новые.

**Адаптивные правила:** брейкпоинты темы 1200/768/575/480; ритм секций 80/56/44; off-canvas-меню
<1200px — не вешать filter/transform на предков.

## 6. Конверсионные улучшения без изменения текста

1. **Sticky mobile CTA-полоса** (пересмотреть CNV-03). Внизу экрана на mobile — полоса
   `📞 Позвонить | WhatsApp` (золото/ink). Согласовать с `.pfg-whatsapp-float` (не накладывать:
   либо полоса заменяет float, либо float уходит над полосу). Самый высокий ROI для трафика.
2. **Телефон+WhatsApp в мобильной шапке** (D1) — иконки 44px всегда видимы.
3. **Повтор primary-CTA по длине страницы** (D9): после hero, после услуг, после процесса,
   в футере — один и тот же золотой «Получить консультацию» (копия не меняется).
4. **Строка/чипы доверия** из реальных фактов: с 2013 · 1С · по законам РК · конфиденциально ·
   договор. Под hero и в about. Без выдумок.
5. **Лид-форма на лендинге** (D5/вариант C): компактная форма (имя+телефон+кнопка) в hero или
   отдельной секцией перед футером, та же логика → WhatsApp (`js/custom.js`, поведение не менять).
6. **Тарифы доступнее:** ссылка-якорь на блок тарифов из hero/услуг; «Популярный» уже выделен.
   Под CTA тарифа — микрокопия доверия (если согласуют новый текст — иначе только визуал).
7. **Микроанимации (сдержанно):** hover-подъём кнопок/карточек (есть), reveal-на-скролл для
   секций (AOS уже подключён), золотое подчёркивание в навигации (есть). Не добавлять лишнего —
   избыток = «ИИ-сгенерированный» вид.
8. **Прогрессия внимания:** hero (1 действие) → доверие → услуги → доказательство → процесс →
   повтор-CTA → форма → футер-CTA. Один золотой акцент на экран, ведёт глаз к заявке.
9. **Новые элементы — только на существующем тексте** или с пометкой «нужен новый текст —
   согласовать» (счётчики, отзывы, гарантия, микрокопия).

## 7. Адаптив и production-ready

**Брейкпоинты:** desktop ≥1201 (стекло-шапка, hero-сплит), tablet 768–1200 (1–2 кол, шапка с
телефоном), mobile ≤575 (1 кол, sticky-CTA, шапка лого+📞+WA+меню). Проверка на 1440/1024/768/390/360.

**Скорость загрузки:**
- Фото hero/og — WebP/AVIF + `loading=lazy` (кроме LCP-hero: `fetchpriority=high`, без lazy).
- Размеры под вьюпорт (`srcset`), сжатие; убрать неиспользуемые плагины темы со страниц, где
  их нет (countdown, charts, magnific без триггеров) — осторожно, без поломки `scripts.js`.
- `font-display:swap` для веб-шрифтов; preconnect Google Fonts.

**Читаемость:** мера 60ch (есть), lh 1.6, контраст AA (есть). H1 — один на страницу (есть).

**Кликабельные зоны:** все интерактивы ≥44×44 (есть для шапки/футера/чекбокса; распространить
на новые mobile-CTA и иконки контакта).

**Accessibility:** focus-visible везде, aria-label на иконках-кнопках, `role=status aria-live`
на форме (есть), reduced-motion точечный (есть), декоративные SVG `aria-hidden`. Цель — не
ухудшить baseline Lighthouse; известный флаг target-size темы (гамбургер↔поиск) — задокументирован.

**Состояния элементов:** hover/focus/active/disabled (кнопки), focus (поля), open (FAQ/аккордеон),
loading/empty/error формы. Все описаны — проверить в макете.

**Sticky-поведение:** sticky-шапка (есть), sticky mobile-CTA (новое) — не перекрывать контент,
учесть safe-area-inset (iPhone).

**SEO-гигиена без текста:** title/description/canonical/OG/JSON-LD не трогать (готовы). Менять
только: `alt` у новых/заменённых фото (описательные), порядок/семантику блоков, `sitemap.xml`
при добавлении страниц. Один H1/стр.

**Передача разработчику:** Figma с токенами (цвет/тайпскейл/spacing/shadow = `--pfg-*`),
компоненты (кнопки/карточки/форма/чипы/таймлайн/sticky-CTA) со всеми состояниями, макеты
3 брейкпоинтов для index + шаблон услуги + contacts, спека анимаций, экспорт ассетов (WebP),
redline-отступы по 4px-сетке. Явно пометить: «текст финальный — не редактировать».

## 8. Финальное ТЗ для дизайнера

1. **Цель редизайна.** Превратить премиальный, но «витринный» сайт в перформанс-лендинг под
   платный трафик: сильный первый экран, видимый контакт на mobile, доверие как визуальный слой,
   сквозная хореография одного CTA → рост заявок (звонок/WhatsApp/форма). Текст не менять.

2. **Что нельзя менять.** Любой текст (заголовки, офферы, кнопки, подписи, FAQ); бренд-палитру
   (ink `#16222d`, gold `#ecab23`); логику форм→WhatsApp, аналитику, JSON-LD, роутинг; вендорную
   тему (правки только `custom.css`/`base.css`/`custom.js`); no-build. Не вводить выдуманные
   цифры/отзывы/логотипы — только реальные факты или резерв-слоты.

3. **Дизайн-концепция.** «Спокойная дорогая бухгалтерия». Ink+gold на бумаге, леджерная
   сигнатура (золотой штрих), hairline-табличный ритм, золото только на действиях. Эволюция
   существующей системы `--pfg-*`, не ре-скин.

4. **Первый экран.** Оффер+заголовок вместе слева (не в углу), золотой primary-CTA в hero,
   строка доверия из реальных фактов, фото — KZ-контекст с ink-дуотоном, единый оффер/CTA на
   всех слайдах (или статичный hero). 3 варианта на тест (A/B/C из раздела 4). Mobile: шапка с
   📞+WA, видимый оффер, CTA во всю ширину, sticky-CTA снизу.

5. **Каждый блок.** По разделу 3: шапка (mobile-контакт), полоса доверия (карточная, 2×2 mobile),
   about (чипы «с 2013/1С»), услуги (чинить обрезку заголовков, ghost-ссылки), marquee (ink, не
   трогать), почему-мы (золотые галочки, равные фото-карточки), процесс (таймлайн 01–04 +
   повтор-CTA), резерв-доверие (слоты), футер (CTA-заголовок+кнопка), услуги-страницы (единый
   шаблон с формой), контакты (форма+блок доверия).

6. **UI-kit.** Раздел 5: цвета-роли, тайпскейл clamp, 3 тира кнопок + состояния, карточки,
   формы, бейджи/чипы, тени/радиусы/отступы (4px), дуотон-фото, сигнатура. Всё на токенах `--pfg-*`.

7. **Адаптив.** Раздел 7: 5 брейкпоинтов, ритм 80/56/44, тап ≥44px, sticky-CTA с safe-area,
   состояния, WebP/lazy/LCP, AA-floor.

8. **Конверсия.** Раздел 6: sticky mobile-CTA, контакт в шапке, повтор CTA ×4, чипы доверия,
   лид-форма на лендинге, доступные тарифы, сдержанные микроанимации, один акцент на экран.

9. **Продакшен-качество.** Реализуемо в `custom.css`/`custom.js` без сборки; правки шапки/
   футера — одинаково во всех 11 HTML; не ломать off-canvas-меню (<1200px без filter/transform
   на предках); Lighthouse не ниже baseline; макет + токены + компоненты + redline для разработчика.

## 9. Чек-лист приёмки дизайна

**Контент/ограничения**
- [ ] Ни одно слово текста не изменено (заголовки, офферы, кнопки, FAQ, подписи).
- [ ] Палитра не расширена сверх ink/gold/paper/hairline; золото только на действиях/1 акценте.
- [ ] Нет выдуманных цифр/отзывов/логотипов; доверие — реальные факты или резерв-слоты.

**Первый экран**
- [ ] Заголовок+оффер читаются вместе, ничего не обрезано (desktop и mobile).
- [ ] В hero есть золотой primary-CTA; вторичный — ghost.
- [ ] Единый оффер/CTA на всех слайдах (или согласован статичный hero); ≥1 вариант на тест.
- [ ] Строка доверия из реальных фактов присутствует.
- [ ] Фото — KZ-контекст с единым ink-дуотоном, текст контрастен (AA).

**Мобильная конверсия**
- [ ] В мобильной шапке видимы 📞 и WhatsApp (≥44px), не спрятаны в меню.
- [ ] Sticky mobile-CTA снизу, согласована с WhatsApp-float, учитывает safe-area.
- [ ] Primary-CTA повторяется по длине страницы (≥4 точек).

**Блоки/иерархия**
- [ ] Один заметный primary-CTA на экран; вторичные не спорят с золотом.
- [ ] Заголовки услуг не обрезаются на 1024/768/390.
- [ ] Процесс 01–04 — таймлайн с золотой связью; нумерация = реальный порядок.
- [ ] Лид-форма доступна на лендинге (hero/секция), а не только на /contacts.

**UI-система**
- [ ] Все интерактивы имеют hover/focus-visible/active/disabled.
- [ ] Поля формы 16px, золотой фокус, чекбокс золото, статус ⚠/✓ (не только цвет).
- [ ] Кнопки radius 8, карточки 14; отступы по 4px-сетке; тени по шкале.
- [ ] Eyebrow/кнопки/меню `text-transform:none` (кириллица не ломается).

**Адаптив/доступность/продакшен**
- [ ] Проверено на 1440/1024/768/390/360 — нет гориз. скролла, переносов, обрезок.
- [ ] Все тап-зоны ≥44px; контраст AA; focus-visible; reduced-motion учтён.
- [ ] LCP-фото оптимизировано (WebP, fetchpriority), остальное lazy.
- [ ] Реализуемо в `custom.css`/`custom.js` без сборки; off-canvas-меню не сломано.
- [ ] Правки шапки/футера консистентны для всех 11 страниц.
- [ ] Lighthouse (perf/a11y/SEO) не ниже текущего baseline.
- [ ] Передан макет + токены + компоненты со состояниями + redline + ассеты.
