# План разработки сайта taxpfg.kz (PrimeFinance Group)

> Статус: контент на согласовании у клиента. Разработку начинаем **параллельно**,
> на базе шаблона GudFin. Тексты черновые — верстаем модульно, чтобы блоки можно
> было легко добавлять/убирать по правкам клиента.
> Источники: `docs/SITE_CONTENT.md` (полный контент), `docs/TEMPLATE_ANALYSIS.md`
> (разбор шаблона), `docs/DRAFT.md` (исходник). Этот файл — рабочий план; ему
> следуем строго и отмечаем прогресс.

## 0. Принципы и решения

1. **Статичный сайт**, без сборки/npm. Многостраничный. Header/footer дублируются
   в каждом файле (подход самой темы) — держим их синхронными по «эталону».
2. **Язык — русский** (`lang="ru"`), вся «рыба» и английский заменяются.
3. **Ничего не выдумываем**: блоки с неподтверждёнными данными (команда, отзывы,
   цифры/статистика, кейсы) **не публикуем** — резервируем место и прячем до данных.
4. **Без ложных обещаний** (нет «нулевых налогов», «без проверок» и т.п.).
5. **Изменяемость**: каждая секция самостоятельна; правки клиента = добавить/убрать
   блок без переписывания страницы.
6. **Версионирование**: локальный git (без удалёнки, без push) — базовый коммит =
   нетронутый шаблон, дальше атомарные коммиты по шагам (rollback при необходимости).

- Нет битых картинок и ошибок в консоли.

- Кликабельные tel/WhatsApp на мобайле; формы открывают WhatsApp с префиллом.
- Локальный предпросмотр (статический сервер) и визуальная проверка.
